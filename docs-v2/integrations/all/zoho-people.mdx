## Access requirements
| Pre-Requisites | Status | Comment|
| - | - | - |
| Paid dev account | ✅ Not required | Free Zoho account is sufficient for development. |
| Paid test account | ✅ Not required | Free Zoho account is sufficient for testing. |
| Partnership | ❌ Required for production | Partnership with Zoho is required to deploy to production. |
| App review | ✅ Not required | No review process for OAuth apps. |
| Security audit | ✅ Not required | |


## Setup guide

Follow the general [Zoho OAuth setup guide](/integrations/all/zoho) to create your OAuth application. When selecting scopes, make sure to include the People-specific scopes listed in the Common Scopes section below.

## Common Scopes

- **ZohoPeople.forms.ALL** - Access to all People forms
- **ZohoPeople.employee.ALL** - Access to all employee data
- **ZohoPeople.attendance.ALL** - Access to all attendance data
- **ZohoPeople.leave.ALL** - Access to all leave data
- **ZohoPeople.timetracker.ALL** - Access to all time tracker data
- **ZohoPeople.shifts.ALL** - Access to all shift data

## API gotchas

<ZohoSharedGotchas />

### People-specific gotchas
- Zoho People API has region-specific endpoints. You need to use the correct endpoint based on the user's region.
- The People API has rate limits that vary based on your Zoho People plan.
- API rate limits are documented at [Zoho People API Limits](https://www.zoho.com/people/api/api-limits.html).
