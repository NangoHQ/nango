# Overview

To authenticate with Builder.io (private), you will need:
1. **API Key** - A key that grants Nango permission to interact with  Builder.io's (private) APIs resources and services.

This guide will walk you through finding or creating a Private API Keys within Builder.io.

### Prerequisites:

- You must have a registered account with  Builder.io.

### Instructions:

#### Step 1: Finding Your API Domain
1. Different APIs within Builder.io require different domains, i.e `Content API` uses the `cdn.builder.io` domain, for more on this please visit [API Intro](https://www.builder.io/c/docs/api-intro)


#### Step 2: Finding Your API Key
1. **Log in** to your Builder.io dashboard.
2. Select your space and go to the **Space settings** section in the sidebar. 
3. Under **Space Settings**, find the **Private Keys** row and click the **Edit** button.
<img src="/integrations/all/builder-io-private/builder_io_api_key.png" />
3. In the modal, view your existing private API key or create a new one if needed.

<img src="/integrations/all/builder-io-private/generate_api_key.png" />


#### Step 3: Enter credentials in the Connect UI

Once you have the **API Key**:
1. Open the form where you need to authenticate with Builder.io (private).
2. Enter both the **API Key** and **Domain** in the designated fields.
3. Submit the form, and you should be successfully authenticated.

<img src="/integrations/all/builder-io-private/form.png" style={{maxWidth: "450px" }}/>

You are now connected to Builder.io (private).
