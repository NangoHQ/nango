# Overview

To authenticate with Formstack Documents, you need:
1. **API Key** - The key used as the Basic Auth username.
2. **API Secret** - The secret used as the Basic Auth password.

This guide walks through collecting both credentials from your Formstack Documents account. The developer portal still uses the legacy `Webmerge` branding.

### Prerequisites:

- You must have access to a Formstack Documents account.

#### Step 1: Generate your API Key and API Secret

1. Sign in to your Formstack Documents account.
2. In the left-hand menu, click **API Access**.
3. Click **New API Key**.
4. Enter a name for the API key.
5. Click **Create Key**.
6. Copy the generated **API Key** and **API Secret**.

<Note>
  Formstack Documents developer documentation is published at `webmerge.me`, and API requests use the legacy base URL `https://www.webmerge.me/api`.
</Note>

<Note>
  For the Nango provider, these credentials are used to authenticate the connection through Basic Auth. In Formstack Documents itself, document merge and route merge may not require API credentials in every case: accounts can allow merge requests without keys, and logged-in users can sometimes merge without providing them manually. Nango still needs the API Key and API Secret because this provider connects directly to the public API.
</Note>

<Tip>
  Recommended for security: enable **Advanced Settings -> Require API Key for all merge requests (custom integrations only)** in Formstack Documents so custom integration merge requests consistently require API credentials.
</Tip>

#### Step 2: Enter credentials in the Connect UI

Once you have your **API Key** and **API Secret**:
1. Open the form where you need to authenticate with Formstack Documents.
2. Enter the API Key in the **API Key** field.
3. Enter the API Secret in the **API Secret** field.
4. Submit the form to complete authentication.

You are now connected to Formstack Documents.
