## üöÄ Quickstart

Connect to Wise with Nango.

<Steps>
    <Step title="Create the integration">
    In Nango ([free signup](https://app.nango.dev)), go to [Integrations](https://app.nango.dev/dev/integrations) -> _Configure New Integration_ -> _Wise (API Key)_.
    </Step>
    <Step title="Generate your API Token">
    1. Go to [wise.com](https://wise.com) and log in
    2. Select your **Business profile**
    3. Navigate to: **Your Account ‚Üí Integrations and Tools ‚Üí API tokens**
    4. Click **Add new Token**
    5. Complete 2-step verification
    6. Copy the generated token
    </Step>
    <Step title="Authorize Wise">
    Go to [Connections](https://app.nango.dev/dev/connections) -> _Add Test Connection_ -> _Authorize_, then enter your API Token. Later, you'll let your users do the same directly from your app.
    </Step>
    <Step title="Call the Wise API">
    Let's make your first request to the Wise API (get your profiles). Replace the placeholders below with your [secret key](https://app.nango.dev/dev/environment-settings), [integration ID](https://app.nango.dev/dev/integrations), and [connection ID](https://app.nango.dev/dev/connections):
    <Tabs>
        <Tab title="cURL">

            ```bash
            curl "https://api.nango.dev/proxy/v1/profiles" \
              -H "Authorization: Bearer <NANGO-SECRET-KEY>" \
              -H "Provider-Config-Key: <INTEGRATION-ID>" \
              -H "Connection-Id: <CONNECTION-ID>"
            ```

        </Tab>

        <Tab title="Node">

        Install Nango's backend SDK with `npm i @nangohq/node`. Then run:

        ```typescript
        import { Nango } from '@nangohq/node';

        const nango = new Nango({ secretKey: '<NANGO-SECRET-KEY>' });

        const res = await nango.get({
            endpoint: '/v1/profiles',
            providerConfigKey: '<INTEGRATION-ID>',
            connectionId: '<CONNECTION-ID>'
        });

        console.log(res.data);
        ```
        </Tab>
    </Tabs>
    Or fetch credentials with the [Node SDK](/reference/sdks/node#get-a-connection-with-credentials) or [API](/reference/api/connection/get).

    ‚úÖ You're connected! Check the [Logs](https://app.nango.dev/dev/logs) tab in Nango to inspect requests.
    </Step>

    <Step title="Implement Nango in your app">
        Follow our [quickstart](/getting-started/quickstart/embed-in-your-app) to integrate Nango in your app.
    </Step>
</Steps>

## API gotchas

-   **Business Account Required**: Personal API tokens are generated from a Wise Business profile.
-   **Limited API Access**: Personal tokens have more limited API access compared to OAuth credentials.
-   **PSD2 Restrictions (EU/UK)**: Due to PSD2 regulations, you **cannot** fund transfers or view balance statements via API with a personal token in the EU/UK.
-   **Single User**: A personal token authenticates requests for a single Wise.com user only.
-   **When to Use OAuth Instead**: If you're building for end customers or need full API access, you should use OAuth (requires Wise partnership).

### Available with Personal Token

- View profiles
- Create quotes
- Create recipients
- View transfers
- Basic account operations

### Not Available with Personal Token (EU/UK)

- Fund transfers via API
- View balance statements
- Some advanced operations

## Security best practices

<Warning>
Your Wise API token provides direct access to your business account. Handle it with care.
</Warning>

- **Never commit tokens to version control** - Use environment variables or a secrets manager instead
- **Rotate tokens periodically** - Generate new tokens regularly and revoke old ones
- **Monitor API usage** - Review your Wise account activity for unauthorized access
- **Use Nango's secure storage** - Nango encrypts and securely stores your credentials
- **Limit token permissions** - Only grant the scopes your integration requires
- **Revoke unused tokens** - Delete tokens for integrations you no longer use
