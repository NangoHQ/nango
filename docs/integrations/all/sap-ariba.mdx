## Setup guide

### Prerequisites
- SAP Ariba Developer Portal account
- Access to an SAP Ariba realm (test or production)

### Step 1: Register Your Application
1. Log in to the [SAP Ariba Developer Portal](https://developer.ariba.com)
2. Navigate to **Manage > Applications**
3. Click **Create New Application**
4. Fill in the required application details
5. Request API access for your needed APIs (e.g., Operational Reporting, Supplier Data)
6. Wait for auto-approval (typically 12 hours)

### Step 2: Obtain OAuth Credentials
1. Navigate to your application's **Settings**
2. Copy your **OAuth Client ID**
3. Generate and copy your **OAuth Client Secret**
4. Copy your **Application Key (API Key)**

### Step 3: Configure in Nango
Use the credentials obtained above when connecting to SAP Ariba through Nango.

## API gotchas

- SAP Ariba uses a custom OAuth 2.0 grant type `openapi_2lo` instead of standard `client_credentials`.
- All API requests require BOTH an OAuth Bearer token AND an `apikey` header.
- The `realm` parameter is required for most API endpoints and identifies your organization.
- Different regions have different API endpoints (US, EU, Australia). Use the correct region suffix:
  - US: leave empty
  - EU: `-eu`
  - Australia: `.au.cloud`
- Rate limits are strict for reporting APIs (1 req/sec, 2 req/min, 8 req/hour, 40 req/day for job submissions).
- Some APIs are asynchronous and require polling for results via "view templates".
- To refresh the token, simply use `nango.getConnection()` to generate a new set of tokens.
