Replace the placeholders below with your [secret key](https://app.nango.dev/dev/environment-settings), [integration ID](https://app.nango.dev/dev/integrations), and [connection ID](https://app.nango.dev/dev/connections):

            <Tab title="cURL">

                ```bash
                curl "https://api.nango.dev/proxy/v0/me" \
                  -H "Authorization: Bearer <NANGO-SECRET-KEY>" \
                  -H "Provider-Config-Key: <INTEGRATION-ID>" \
                  -H "Connection-Id: <CONNECTION-ID>"
                ```

            </Tab>

            <Tab title="Node">

            ```typescript

            const nango = new Nango({ secretKey: '<NANGO-SECRET-KEY>' });

            const res = await nango.get({
                endpoint: '/v0/me',
                providerConfigKey: '<INTEGRATION-ID>',
                connectionId: '<CONNECTION-ID>'
            });

            console.log(res.data);
            ```
            </Tab>

   <Steps>
  <Step title="Get your Totango account">
    You need a Totango account to use the API. If you don't have one, sign up at [Totango](https://app.totango.com/).
  </Step>
  <Step title="Find your App Token">
    1. Log in to your [Totango account](https://app.totango.com/).
    2. Follow the instructions [here](https://support.totango.com/hc/en-us/articles/203036939-Personal-Access-Token-and-Service-ID) to get your App Token.
    3. Copy the App Token.
  </Step>
  <Step title="Find your Service ID">
    1. In the same Totango support article, you'll also find instructions for getting your Service ID.
    2. Copy the Service ID - it should be in the format `SP-XXXXXX-XX`.
  </Step>
  <Step title="Configure your integration">
    When setting up your Totango integration in Nango, you'll need to provide:
    - **App Token**: Your personal access token from Totango
    - **Service ID**: Your unique service identifier from Totango
  </Step>

  <Tab title="ðŸš¨ API gotchas">.
    - **Service ID Format**: Service IDs must follow the pattern `SP-XXXXXX-XX` where X represents digits. Make sure to include the `SP-` prefix and hyphens.
    - **API Base URL**: All Totango API requests are made to `https://api.totango.com/api/v3`. Nango handles this automatically when using the proxy.
    - **Authentication Header**: Totango uses the `app-token` header for authentication, not the standard `Authorization` header.
    - **Rate Limits**: Be aware of Totango's API rate limits. Check the [API documentation](https://support.totango.com/hc/en-us/articles/360000000006-API-Overview) for current limits.
