# Overview

To authenticate with the Tripletex test environment you need:

1. **Consumer Token** — provided to you by the developer of the integration. It identifies the integration, not your account.
2. **Employee Token** — generated by you inside your Tripletex test tenant. It controls which data the integration can read or write on your behalf.

This guide walks you through generating the employee token within your Tripletex test tenant.

### Prerequisites

- A Tripletex test account. Register for free at [api-test.tripletex.tech](https://api-test.tripletex.tech). For production, use the [Tripletex](/api-integrations/tripletex/connect) integration instead.
- The consumer token and the integration's application name, both supplied by the developer.

### Instructions

<Note>These steps mirror the [production](/api-integrations/tripletex/connect) flow — only the environment differs. Run them in your test tenant at `api-test.tripletex.tech`, where you can self-issue the consumer token from a free developer account. Test-environment tokens do not work in production.</Note>

<a id="step-1-obtain-the-consumer-token"></a>
#### Step 1: Obtain the consumer token

The consumer token is shared once across all customers of an integration. The developer of the integration sends it to you, along with the **application name** registered with Tripletex. When you are building and testing your own integration, create the consumer token from your test account at [api-test.tripletex.tech](https://api-test.tripletex.tech).

<a id="step-2-generate-an-employee-token"></a>
#### Step 2: Generate an employee token

1. Log in to your Tripletex test tenant.
2. Click the avatar in the top-right corner and pick **My profile**.
3. Above the profile pane, click **More ▾** and choose **API access**.

<img src="/api-integrations/tripletex-sandbox/api_access_menu.png" />

4. On the **API employee tokens** page, click **New token** at the bottom of the list.

<img src="/api-integrations/tripletex-sandbox/api_tokens.png" />

5. In the **New employee token** dialog, pick the developer's integration from the **Application** dropdown, or tick **Adapted setup** to configure the token yourself.

<img src="/api-integrations/tripletex-sandbox/new_token.png" />

6. To control exactly what the integration can access, tick **Adapted setup** and select **Manual set-up**. This grants granular, per-module entitlements instead of all-or-nothing access.

<img src="/api-integrations/tripletex-sandbox/manual_setup.png" />

7. Tripletex groups entitlements by module (general access, time sheet, invoicing, accounting, salary, and more). Enable only the scopes the integration needs.

<img src="/api-integrations/tripletex-sandbox/entitlements.png" />

8. Give the token a memorable name and click **Create token**.

<img src="/api-integrations/tripletex-sandbox/new_token_filled.png" />

   <Warning>If you see *"The application could not be found"*, the application name does not match what the developer registered with Tripletex. The name is case-sensitive and must match exactly.</Warning>

9. Tripletex shows the generated employee token in a one-time popup. **Copy it immediately** — it cannot be recovered. If you misplace it, generate a new one (which invalidates the old one).

<a id="step-3-set-the-proxy-company-id"></a>
#### Step 3: Set the proxy company ID

Leave this at `0` unless the integration developer specifically told you otherwise. A non-zero value is only required for accountant-token flows where one token spans multiple client tenants.

<a id="step-4-enter-credentials-in-the-connect-ui"></a>
#### Step 4: Enter credentials in the Connect UI

Once you have your **Consumer Token**, **Employee Token**, and **Proxy Company ID**:

1. Open the form where you need to authenticate with the Tripletex test environment.
2. Enter your **Consumer Token**, **Employee Token**, and **Proxy Company ID** in their respective fields.
3. Submit the form, and you should be successfully authenticated.

<img src="/api-integrations/tripletex-sandbox/form.png" style={{maxWidth: "450px" }}/>

You are now connected to the Tripletex test environment.
