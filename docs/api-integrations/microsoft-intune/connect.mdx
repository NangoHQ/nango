# Overview

To authenticate with Microsoft Intune, you need:
1. **Tenant ID** — Your organization's unique Microsoft directory identifier.
2. **Client ID** — The application ID assigned to your Entra ID app registration.
3. **Client Secret** — A secret generated for your app registration.

Intune is accessed via the Microsoft Graph API using application permissions (client credentials). There is no delegated flow — an admin must register an app and grant it Intune scopes directly.

### Prerequisites

- An active Azure account with a Microsoft Intune subscription.
- Permission to register applications and grant admin consent in Microsoft Entra ID.

### Instructions

#### Step 1: Register an application in Microsoft Entra ID

1. Go to the [Microsoft Entra admin center](https://entra.microsoft.com) and sign in as an admin.
2. Navigate to **Identity** > **Applications** > **App registrations**.
<img src="/integrations/all/microsoft-business-central/app_registration.png" />
3. Select **New registration**.
<img src="/integrations/all/microsoft-business-central/new_registration.png" />
4. In the **Register an application** section, enter a name for the application (e.g. `Nango Intune`). Under **Supported account types**, select **Accounts in any organizational directory (Multitenant)**. Select **Register**.
<img src="/integrations/all/sharepoint-online-oauth2-cc/register_new_app.png" />
5. On the app's **Overview** page, copy the **Application (client) ID** — this is your **Client ID** — and the **Directory (tenant) ID** — this is your **Tenant ID**.
<img src="/integrations/all/sharepoint-online-oauth2-cc/client_id.png" />

#### Step 2: Add Intune API permissions

1. In your app registration, go to **API permissions** > **Add a permission**.
2. Select **Microsoft Graph** > **Application permissions**.
3. Search for and add the Intune scopes your integration needs. Common scopes:
   - `DeviceManagementManagedDevices.Read.All` — read managed devices
   - `DeviceManagementManagedDevices.ReadWrite.All` — read and write managed devices
   - `DeviceManagementConfiguration.Read.All` — read device configuration policies
   - `DeviceManagementApps.Read.All` — read mobile app management data
4. Select **Add permissions**.
5. Select **Grant admin consent for [your organization]** and confirm. Admin consent is required for application permissions.
<img src="/api-integrations/microsoft-oauth2-cc/permissions.png" />

<Note>
The Intune service must be correctly licensed on the tenant for these API calls to succeed, regardless of whether the permissions are granted.
</Note>

#### Step 3: Create a client secret

1. In your app registration, go to **Certificates & secrets** > **New client secret**.
2. Enter a description and choose an expiry, then select **Add**.
<img src="/integrations/all/sharepoint-online-oauth2-cc/client_secret.png" />
3. Copy the secret **Value** immediately — it is shown only once.
<img src="/integrations/all/sharepoint-online-oauth2-cc/gen_client_secret.png" />

#### Step 4: Enter credentials in the Connect UI

1. Open the form where you need to authenticate with Microsoft Intune.
2. Enter your **Tenant ID**, **Client ID**, and **Client Secret**.
3. Submit the form to authenticate.

<img src="/api-integrations/microsoft-oauth2-cc/form.png" style={{maxWidth: "450px" }}/>

You are now connected to Microsoft Intune.
