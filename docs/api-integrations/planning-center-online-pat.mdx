## 🚀 Quickstart

Connect to Planning Center Online (PAT) with Nango.

<Steps>
    <Step title="Create the integration">
        In Nango ([free signup](https://app.nango.dev)), go to [Integrations](https://app.nango.dev/dev/integrations) -> _Configure New Integration_ -> _Planning Center Online (Personal Access Token)_.

        Personal Access Tokens are intended for **single-user scripting**. If you’re distributing your integration to **multiple churches**, use OAuth instead:

        - [Planning Center Online (OAuth)](/api-integrations/planning-center-online)
    </Step>
    <Step title="Create a Personal Access Token">
        Follow Planning Center’s PAT instructions:

        - [Personal Access Token](https://api.planningcenteronline.com/docs/overview/authentication#personal-access-token)

        When you generate a token, Planning Center gives you a **Client ID** (`client_id`) and **Secret** (`secret`). In Nango Connect, enter:

        - **Client ID**: your PAT `client_id`
        - **Secret**: your PAT `secret`

        These map to HTTP Basic Authentication (`Authorization: Basic ...`), equivalent to `curl -u client_id:secret`.
    </Step>
    <Step title="Call the Planning Center Online API">
        Example (People API): list people records.

        Replace the placeholders below with your [secret key](https://app.nango.dev/dev/environment-settings), [integration ID](https://app.nango.dev/dev/integrations), and [connection ID](https://app.nango.dev/dev/connections):

        <Tabs>
            <Tab title="cURL">

                ```bash
                curl "https://api.nango.dev/proxy/people/v2/people?per_page=1" \
                  -H "Authorization: Bearer <NANGO-SECRET-KEY>" \
                  -H "Provider-Config-Key: <INTEGRATION-ID>" \
                  -H "Connection-Id: <CONNECTION-ID>"
                ```

            </Tab>

            <Tab title="Node">

                Install Nango's backend SDK with `npm i @nangohq/node`. Then run:

                ```typescript
                import { Nango } from '@nangohq/node';

                const nango = new Nango({ secretKey: '<NANGO-SECRET-KEY>' });

                const res = await nango.get({
                    endpoint: '/people/v2/people',
                    params: { per_page: 1 },
                    providerConfigKey: '<INTEGRATION-ID>',
                    connectionId: '<CONNECTION-ID>'
                });

                console.log(res.data);
                ```

            </Tab>

        </Tabs>

        Or fetch credentials with the [Node SDK](/reference/sdks/node#get-a-connection-with-credentials) or [API](/reference/api/connection/get).

        ✅ You're connected! Check the [Logs](https://app.nango.dev/dev/logs) tab in Nango to inspect requests.
    </Step>

    <Step title="Implement Nango in your app">
        Follow our [Auth implementation guide](/guides/primitives/auth) to integrate Nango in your app.
    </Step>
</Steps>

## API gotchas

- PAT uses **HTTP Basic Auth** (`client_id` + `secret`), not bearer tokens.
- PAT is **not appropriate for multi-church integrations** — use OAuth for distributed apps.
- Planning Center requires an identifiable **`User-Agent`**. Nango sends `Nango (https://nango.dev)` by default on proxy and verification requests. To use your own string (for example your app name and support email), set optional connection metadata **`userAgent`** when you create the connection (same mechanism as other integrations’ [connection configuration](/reference/api/connection/post#body-connection-config)).
