# Overview

Planning Center Personal Access Tokens (PATs) allow you to call the API as a single user (great for scripts and internal tooling).

If you need to distribute your integration to **multiple churches**, use OAuth instead:

- [Planning Center Online (OAuth)](/api-integrations/planning-center-online)

### Prerequisites

- You can sign in to Planning Center and generate Personal Access Tokens from your developer account.

## Connect in Nango

<Steps>
  <Step id="open-planning-center-personal-access-tokens" title="Open Personal Access Tokens">
    1. Open Planning Center PAT management: `https://api.planningcenteronline.com/personal_access_tokens`

    Official documentation:

    - [Personal Access Token](https://api.planningcenteronline.com/docs/overview/authentication#personal-access-token)
  </Step>
  <Step id="create-planning-center-personal-access-token" title="Create a personal access token">
    <a id="step-2-create-a-personal-access-token"></a>

    1. Create a new token.
    2. Copy the **Client ID** (`client_id`) and **Secret** (`secret`).

    These credentials authenticate via HTTP Basic Auth (equivalent to `curl -u client_id:secret ...`).
  </Step>
  <Step id="enter-planning-center-pat-credentials-in-connect" title="Enter credentials in Nango Connect">
    In the Connect UI form:

    - **Client ID**: paste your PAT `client_id`
    - **Secret**: paste your PAT `secret`

    Submit the form to complete authentication.
  </Step>
</Steps>
