This quickstart uses the GitHub API as an example, but you can choose any of Nango's [700+ supported APIs](/integrations/overview).

<Steps>
  <Step title="Create a GitHub integration">
    [Sign up](https://app.nango.dev/signup) for free, then go to the Integrations tab > Set up new integration > GitHub (User OAuth).
  </Step>

  <Step title="Authorize GitHub">
    Go to the Connections tab > Add Test Connection > pick your GitHub integration, and complete the auth flow.

    Copy the integration ID, connection ID, and your API key in the Environment settings tab > API Keys. You will use them in the next step.

    <Accordion title="For agents">
      To create a browser auth link programmatically, call:

      ```bash
      curl --request POST \
        --url https://api.nango.dev/connect/sessions \
        --header 'Authorization: Bearer <NANGO-SECRET-KEY>' \
        --header 'Content-Type: application/json' \
        --data '{
          "tags": { "end_user_id": "quickstart-user" },
          "allowed_integrations": ["<INTEGRATION-ID>"]
        }'
      ```

      Send the returned `connect_link` to the user, then confirm the connection in the dashboard.
    </Accordion>
  </Step>

  <Step title="Enable an action function">
    Open your GitHub integration and enable the `get-repository` template action function.

    This is a Nango Function: it runs provider-specific code with the connected GitHub account's credentials, without exposing those credentials to your app or agent.
  </Step>

  <Step title="Call the action">
    Trigger the function from your backend:

    <Tabs>
      <Tab title="Node">
      Install the SDK with `npm i @nangohq/node`, then run:

      ```typescript
      import { Nango } from '@nangohq/node';

      const nango = new Nango({ secretKey: process.env.NANGO_SECRET_KEY! });

      const result = await nango.triggerAction(
          '<INTEGRATION-ID>',
          '<CONNECTION-ID>',
          'get-repository',
          {
              owner: 'NangoHQ',
              repo: 'nango'
          }
      );

      console.log(result);
      ```
      </Tab>

      <Tab title="cURL">
      ```bash
      curl --request POST \
        --url https://api.nango.dev/action/trigger \
        --header 'Authorization: Bearer <NANGO-SECRET-KEY>' \
        --header 'Connection-Id: <CONNECTION-ID>' \
        --header 'Provider-Config-Key: <INTEGRATION-ID>' \
        --header 'Content-Type: application/json' \
        --data '{
          "action_name": "get-repository",
          "input": {
            "owner": "NangoHQ",
            "repo": "nango"
          }
        }'
      ```
      </Tab>
    </Tabs>

    You should receive GitHub repository details such as the repository ID, full name, visibility, default branch, and owner.
  </Step>

  <Step title="Inspect the run">
    Open the [Logs](https://app.nango.dev/dev/logs) tab to inspect the action execution, provider request, response, and any errors.

    🎉 You connected an API and ran your first integration function.
  </Step>
