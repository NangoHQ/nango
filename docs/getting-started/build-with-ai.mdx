Use Nango's public docs MCP server and skills to give your coding agent current Nango context while it builds integrations.

<a id="docs-mcp-server"></a>
## Docs MCP server

Add Nango's documentation MCP server to your coding agent:

```text
https://nango.dev/docs/mcp
```

It is public and does not require authentication.

<a id="skills"></a>
## Skills

Install the Nango skills package:

```bash
npx skills add NangoHQ/skills
```

The skills teach agents how to create, test, and deploy Nango functions.
